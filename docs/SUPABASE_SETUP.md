# Supabase Configuration Guide

This guide provides instructions for setting up Supabase, creating tables, enabling Row Level Security (RLS), and configuring API endpoints.

## 1. Setting Up Supabase
1. Go to the [Supabase website](https://supabase.io) and sign in or create an account.
2. Create a new project by clicking on the `New Project` button.
3. Fill in your project details including the project name, password, and select a region.
4. Click on `Create new project` and wait for the setup to complete.

## 2. Creating Tables
1. In the Supabase dashboard, navigate to the `Table Editor`.
2. Click on `New Table`.
3. Define your table name and columns. For example:
   - **Table Name:** reviews
   - **Columns:**
     - id (integer, primary key)
     - user_id (uuid)
     - review_text (text)
     - created_at (timestamp, default: `now()`)
4. Click on `Save` to create the table.

## 3. Enabling Row Level Security (RLS)
1. Go to the `Authentication` section in the sidebar.
2. Enable RLS for your project.
3. Navigate back to the `Table Editor` and select the table you want to secure (e.g., `reviews`).
4. Click on the `RLS` tab and toggle the `Enable Row Level Security` option.
5. Define specific policies by clicking on `New Policy` and specifying the conditions based on your requirements (e.g., allow users to view only their reviews).

## 4. Configuring API Endpoints
1. Supabase automatically generates a RESTful API for your tables. Navigate to the `API` section in the sidebar.
2. You will find the base URL for your project and the documentation for the autogenerated API endpoints.
3. Use the provided endpoints to interact with your tables, such as:
   - `GET /rest/v1/reviews` to fetch reviews.
   - `POST /rest/v1/reviews` to add a new review.
4. Ensure that your API keys are correctly set up for authentication and access control.

## Conclusion
Following this guide will help you set up your Supabase backend, create tables, enable security features, and configure API endpoints. For further information, refer to the [Supabase documentation](https://supabase.io/docs).
